Architecture ‚Äî Auth Refactoring

Overview

This document describes the architecture for migrating from session-based authentication to JWT tokens with OAuth2 provider support. The design prioritizes backward compatibility during migration and zero-downtime deployment.

Token Architecture

The token service uses asymmetric key pairs (RS256) for JWT signing. Access tokens are short-lived (15 minutes) with refresh tokens stored in HTTP-only cookies.

OAuth2 Flow

We use the authorization code flow with PKCE for all OAuth2 providers.

type AuthProvider interface {
  AuthURL(state string) string
  Exchange(code string) (*TokenPair, error)
  UserInfo(token string) (*UserProfile, error)
}
